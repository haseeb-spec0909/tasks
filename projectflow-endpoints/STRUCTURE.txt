Authentication & Security
========================

All endpoints require:
1. API_KEY header (validated via validateApiKey or validateTimeIntelApiKey)
2. Client IP in allowed range (default: Cloud Run ranges + localhost)
3. Valid HTTP method (GET vs POST)
4. Valid parameter format and ranges

Database Security
================

- OCI8 prepared statements with bind variables (prevents SQL injection)
- Manual transaction control (BEGIN/COMMIT/ROLLBACK)
- Proper connection cleanup
- Error logging without exposing sensitive data

Transaction Flow (updateTaskProgress)
===================================

1. BEGIN TRANSACTION (Manual commit mode)
2. Fetch current task data (validation)
3. UPDATE TMC_WORKPACKAGE_TASK SET progress = X
4. Call updateWPActualProgress(wp_id)
5. Call updateDeliverableActualProgress(del_id)
6. Call updateBuildActualProgress(build_id)
7. Call updateProjectActualProgress(project_id)
8. Call saveStatusHistory()
9. Call saveDateHistory()
10. COMMIT (success) or ROLLBACK (any error)

Configuration
=============

In your main ProjectFlow config file, set:

    $GLOBALS['API_KEY'] = 'your-secret-api-key';
    $GLOBALS['TIMEINTEL_API_KEY'] = 'timeintel-key-optional';
    $GLOBALS['DB_USER'] = 'oracle_user';
    $GLOBALS['DB_PASS'] = 'oracle_password';
    $GLOBALS['DB_NAME'] = 'oracle_tns_alias';

Installation
============

1. Copy webServices/ directory to ProjectFlow's _webServices/
2. Add configuration to ProjectFlow main config file
3. Update IP allowlist in api_auth.php if needed
4. Test with: GET /webServices/healthcheck.php
5. Configure rate limiting (100 req/min recommended)

Testing
=======

Health Check:
  curl -H "API_KEY: key" https://projectflow/webServices/healthcheck.php

Get Tasks:
  curl -H "API_KEY: key" "https://projectflow/webServices/getMyTasks.php?user_id=42"

Update Progress:
  curl -X POST -H "API_KEY: key" -H "Content-Type: application/json" \
    -d '{"wp_task_id":1001,"progress_pct":75,"updated_by_user_id":42}' \
    https://projectflow/webServices/updateTaskProgress.php

Logging
=======

All API calls logged to SYS_EVENT_LOG:
- EVENT_TYPE: TIMEINTEL_API_CALL
- Details: JSON with endpoint, user_id, client_ip, parameters
- Level: INFO

Query: SELECT * FROM SYS_EVENT_LOG WHERE EVENT_TYPE = 'TIMEINTEL_API_CALL' ORDER BY CREATED_DT DESC;
