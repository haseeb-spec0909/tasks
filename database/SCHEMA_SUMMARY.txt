KEY FEATURES
================================================================================

1. MULTI-SOURCE TASK MANAGEMENT
   - Sync from Google Tasks and ProjectFlow
   - Automatic deduplication (UNIQUE on source+external_id+user_id)
   - Full audit trail via sync_log

2. INTELLIGENT SCHEDULING
   - Google Calendar integration with real-time Push API support
   - Focus time management with flexible/locked modes
   - Time-block based scheduling with split support
   - Free-slot availability calculation

3. PROJECTFLOW INTEGRATION
   - Complete WBS hierarchy tracking (pf_task_meta)
   - Deliverable, build, and project context
   - Baseline date tracking
   - Automatic priority calculation

4. USER CUSTOMIZATION
   - Per-user working hours (JSON by day)
   - Configurable focus targets (minutes/day)
   - Notification preferences (JSON)
   - Timezone support (default Asia/Karachi)
   - Email domain enforcement (@tmcltd.ai)

5. REAL-TIME SYNCHRONIZATION
   - Google Calendar push notifications
   - ProjectFlow polling with error tracking
   - Checksum-based change detection
   - Consecutive error counting for backoff

6. HABIT TEMPLATES
   - Global templates for team-wide adoption
   - User-specific instances with inheritance
   - Recurrence rules (RFC 5545 compatible)
   - Priority and duration constraints

CONSTRAINTS & VALIDATION
================================================================================

Domain Constraints:
  - Email domain must be @tmcltd.ai (CHECK constraint)
  - Role: consultant | manager | admin
  - Source: GOOGLE_TASKS | PROJECTFLOW | GOOGLE_CALENDAR
  - Direction: INBOUND | OUTBOUND
  - Status: 7 valid values (NOT_STARTED through CANCELLED)
  - Priority: P1_CRITICAL | P2_HIGH | P3_MEDIUM | P4_LOW
  - Block type: task | focus | habit
  - Focus defense: flexible | locked
  - Event category: 8 categories (one_on_one through other)

Data Integrity:
  - Progress percentage: 0-100 (CHECK)
  - Consecutive errors: >= 0 (CHECK)
  - Scheduling state: must have task_id OR habit_id (CHECK)
  - Task uniqueness: (source, external_id, user_id)
  - Cache uniqueness: (user_id, gcal_event_id)
  - Team uniqueness: (manager_user_id, member_user_id)

Cascade Behavior:
  - User deletion → cascades to all related tables
  - Task deletion → cascades to pf_task_meta, scheduling_state, notifications
  - Habit deletion → cascades to scheduling_state
  - Owner deletion → set to NULL for orphaned tasks

PERFORMANCE CHARACTERISTICS
================================================================================

Critical Path Indexes (Tier 1):
  - idx_tasks_user_status: User dashboard queries
  - idx_sched_time_range: Availability calculations
  - idx_cache_time_range: Calendar scheduling
  - idx_pf_meta_task_id: ProjectFlow metadata joins

Common Query Indexes (Tier 2):
  - idx_tasks_user_source: Source-specific task lists
  - idx_sync_user_source_time: Audit history
  - idx_notif_user_id + sent_at: Recent notifications
  - idx_habits_user_active: Active habit scheduling

Lookup Indexes (Tier 3):
  - idx_users_google_id: OAuth lookups
  - idx_pf_meta_wp_task_id: ProjectFlow task mapping
  - idx_team_manager_id: Manager team lookup

Write Optimization:
  - Append-only sync_log design
  - UNIQUE constraints support ON CONFLICT semantics
  - Denormalized fields (wp_code) reduce joins

Query Performance:
  - User task queries: ~1ms with index
  - Availability slot finding: 5-50ms depending on calendar density
  - Sync audit queries: <1ms with composite index
  - Calendar event lookups: <1ms with time-range index

Storage Estimate (500 users, 1 year):
  - Total with indexes: 3-4 GB
  - sync_log will be largest (875 MB estimated)
  - Annual growth: ~0.5-1 GB per 500 users
